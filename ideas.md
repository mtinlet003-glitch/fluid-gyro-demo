# Fluid Gyro Demo - Design Brainstorm

## Reference Design
ユーザーが提示した「MTINLET」のデザインを参考にします。このデザインの特徴：
- **背景**: ベージュ/クリーム色の落ち着いた色合い
- **流体要素**: 黒い液体が有機的に広がる、ミニマルで高級感のあるビジュアル
- **タイポグラフィ**: 大きく太い見出し、シンプルで洗練されたレイアウト
- **インタラクション**: カーソルの動きに応じて流体が反応する
- **全体的な雰囲気**: モダン、エレガント、テクノロジーと自然の融合

## 選定デザイン方向

### Design Movement
**Organic Minimalism with Digital Fluidity** - 自然界の流体現象とデジタル技術の融合

### Core Principles
1. **有機的な流動性**: 液体の動きは自然で予測不可能、しかし計算された美しさ
2. **ミニマルな構成**: 必要最小限の要素で最大の視覚的インパクト
3. **高級感**: 落ち着いた色合い、丁寧なスペーシング、洗練された動き
4. **インタラクティブ性**: ユーザーの行動（ジャイロ）が直接ビジュアルに反映

### Color Philosophy
- **背景**: `#f5f1ed` - ウォームなベージュ（自然、落ち着き）
- **液体**: `#1a1a1a` - 深い黒（コントラスト、高級感）
- **アクセント**: `#d4af37` - 金色の微かなハイライト（洗練、価値）
- **テキスト**: `#2c2c2c` - ほぼ黒（読みやすさ、統一感）

### Layout Paradigm
- **非中央集中型**: 液体シミュレーションが画面全体を支配、テキストは周辺に配置
- **非対称性**: 左右のバランスを意図的に崩す（有機的な流れを表現）
- **ビューポート最大活用**: Three.jsキャンバスが画面全体を占める

### Signature Elements
1. **流体シミュレーション**: 黒い液体が画面上を流れ、ジャイロセンサーに反応
2. **グラデーション効果**: 液体の端に微かな発光、深度感を演出
3. **タイポグラフィオーバーレイ**: キャンバス上に浮かぶテキスト、半透明の背景

### Interaction Philosophy
- **ジャイロ連動**: スマートフォンを傾けると液体が重力に従って流れる
- **スムーズな物理演算**: 加速度、減衰、衝突を自然に表現
- **フィードバック**: 液体の動きそのものがユーザーの行動への応答

### Animation
- **液体の流れ**: 60fps、物理ベースのシミュレーション
- **エントランス**: ページ読み込み時、液体が徐々に画面に満ちる（1.5秒）
- **ホバー効果**: マウス/タッチポイントで液体が引き寄せられる（オプション）
- **テキストフェード**: 初期表示時、テキストが液体の上にフェードイン

### Typography System
- **見出し**: `Playfair Display` (bold, 3.5rem) - 優雅で存在感のある大きさ
- **本文**: `Inter` (regular, 1rem) - 読みやすく、モダン
- **ラベル**: `Inter` (medium, 0.875rem) - 指示文やメタ情報

### Brand Essence
**"Motion meets minimalism"** - テクノロジーと自然の融合を通じて、デジタルの可能性を表現する。
- **3つの性格**: エレガント、インタラクティブ、革新的

### Brand Voice
- **見出し**: 「流体のように、自由に。」「傾けて、感じる。」
- **説明文**: 「スマートフォンの傾きで液体が流れる。自然な物理演算と高度なシェーダー技術が融合した、没入型のビジュアル体験。」
- **CTA**: 「デバイスを傾けて始める」

### Wordmark & Logo
- **ロゴ**: 流動する液体の形を抽象化した、シンプルな記号
- **フォント**: ロゴテキストなし、シンボルのみ（透明背景のPNG）

### Signature Brand Color
**#1a1a1a** - 深い黒。液体の色として、このブランドの中核を成す。

## 実装方針

### フェーズ1: 基本構造
- Three.jsシーン、カメラ、レンダラーの初期化
- WebGLシェーダー（頂点・フラグメント）の実装
- ジャイロセンサーAPI（DeviceOrientation）の統合

### フェーズ2: 流体シミュレーション
- 粒子ベースまたはグリッドベースのシミュレーション
- 重力、粘性、圧力の物理演算
- パフォーマンス最適化（モバイル対応）

### フェーズ3: UI・テキスト
- ブランドテキストのオーバーレイ
- ジャイロ許可ダイアログ
- レスポンシブデザイン対応

### フェーズ4: ポーランド・最適化
- アニメーション調整
- パフォーマンスプロファイリング
- クロスブラウザ・クロスデバイステスト
